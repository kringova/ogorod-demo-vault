# Сценарии использования personal-site

Ключевые пути, которыми аудитория добивается результата. По ним проходят smoke-тесты:
каждый сценарий ссылается на покрывающий тест; непокрытый помечен ⚠.

## S1: Читатель открывает пост из блога

- **Кто:** случайный посетитель по ссылке из соцсети
- **Цель (JTBD):** прочитать конкретный пост, не потерявшись в навигации
- **Шаги:** 1) переходит по ссылке 2) видит заголовок, дату, текст поста 3) может вернуться к списку постов
- **Ожидаемый результат:** пост открывается быстро, читаем на телефоне, есть путь назад
- **Smoke:** `e2e/blog-post.spec.ts`

## S2: Автор публикует новый пост

- **Кто:** я сам
- **Цель (JTBD):** когда дописал пост, хочу опубликовать его одной командой
- **Шаги:** 1) кладу markdown-файл в `content/posts/` 2) `git push` 3) проверяю деплой
- **Ожидаемый результат:** пост появляется на сайте в течение минуты, попадает в RSS
- **Smoke:** ⚠ нет покрытия — проверяю руками после каждого деплоя

## S3: Рекрутер просматривает портфолио с телефона

- **Кто:** рекрутер или потенциальный клиент
- **Цель (JTBD):** быстро понять, чем я занимаюсь, и найти способ написать
- **Шаги:** 1) открывает сайт с телефона 2) листает карточки проектов 3) находит контакты
- **Ожидаемый результат:** страница не разъезжается, контакты кликабельны (mailto/tel)
- **Smoke:** `e2e/mobile-portfolio.spec.ts`
